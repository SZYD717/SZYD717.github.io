过拟合和欠拟合

发表于 2024-03-07 更新于 2024-03-08 分类于 计算机基础 ， AI 本文字数： 216 阅读时长 ≈ 1 分钟

过拟合和欠拟合是验证损失较大的两个成因。

如果验证损失很小，即使发生了一定的过拟合也是可以被接受的，验证损失小的情况下，如果与训练损失的差值不大，要尝试扩大模型自由度以获得更小的验证损失

当验证损失较大的情况下，训练损失较小，模型很可能发生了过拟合。训练损失如果也较大，和验证损失的差值不大，模型很可能发生了欠拟合。

验证损失小 验证损失大
训练损失小 结合验证损失的绝对值，小概率欠拟合 大概率过拟合
训练损失大 大概率不会发生此类情况 大概率欠拟合

DNS和RDNS

发表于 2023-11-14 更新于 2024-03-08 分类于 计算机基础 ， 互联网协议 本文字数： 2.6k 阅读时长 ≈ 4 分钟

DNS是什么？

《计算机网络：自顶向下方法》的2.4节的标题是一个很好的描述：internet的目录服务 DNS的全称是Domain Name System 域名系统，它负责完成从域名到IP地址的映射。

为什么要有域名/IP地址，只留一个不行吗？

  • IP地址是定长的，利于路由器做规则的处理
  • 域名是变长的，而且很多变。但是往往具备一定人类较容易理解和记忆的规律，例如后缀.fr/.hk/.cn往往表示了该网站的所属的国家和地区 DNS把域名映射到IP地址，这样实际上查询资源的时候用的就是格式统一的IP地址，效率大大的提升了

DNS的结构/实现方式是什么

DNS实现的具体方式：

  • 一个分层的DNS服务器组成的分布式数据库
  • 一个用于让主机查询分布式数据库的应用层（Layer 5）的协议
  • DNS运行在UDP上，使用53号端口
  • DNS一般是运行BIND软件的UNIX系统主机

Network Layer

Network layer的概述

Network层的sender端将来自传输层的segment封装为datagrams，并执行从host到host的传输，receiver端再解包成segment给传输层。Network层的两大作用是Forwarding和Routing Forwarding-转发： 将packets从router的输入送到正确的输出（如何通过当前Router），通常只需要几纳秒 Routing-路由选择：决定packets要到达目标IP该走哪条route（如何到达目的地，要经过哪些Route），通常为几秒 Router作为负责这一层的协调者，会检查每一个通过它的datagram的IP地址 ## Data Plane 从功能上来说，是Forwarding 转发。不涉及路线规划，只负责根据datagram的目标IP传输到正确的输出端口中。 根据现有的路由表和datagram的IP地址来决定转发到哪个端口
